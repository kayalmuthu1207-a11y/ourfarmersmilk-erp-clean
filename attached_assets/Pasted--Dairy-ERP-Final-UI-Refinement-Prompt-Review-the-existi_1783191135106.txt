# Dairy ERP - Final UI Refinement Prompt

Review the existing Dairy ERP UI prototype and business requirements.

This is the FINAL UI REFINEMENT before Database Architecture (Phase 2B) begins.

IMPORTANT:

* UI ONLY
* Do NOT implement backend
* Do NOT implement database

Create only the complete stakeholder-review-ready UI.

---

## 1. Mobile-First Design

The entire ERP must be fully responsive and optimized for:

* Android Phones
* Tablets
* Desktop

Primary users are expected to use mobile devices.

Target Users:

* Owners
* Admins
* Managers
* Accountants
* Delivery Managers
* Delivery Partners
* Customers

All screens must provide a clean mobile experience with easy navigation and minimal clicks.

---

## 2. Customer Portal

Customers must have access to:

* Dashboard
* Milk Orders
* Value Added Product Orders
* Order History
* Invoice History
* Payment History
* Delivery Locations

### Customer Structure

One customer may have:

* Multiple portal users
* Multiple phone numbers
* Multiple delivery locations

Example:

Customer:
Tilaga Enterprises

Locations:

* Unity Enclave
* Green Residency
* Lake View Apartments

The portal should clearly display delivery locations while keeping customer details available.

---

## 3. Milk Ordering Workflow

Create a dedicated Milk Ordering module.

Products may include:

* Cow Milk
* Standardized Milk
* Homogenized Milk
* Branded Milk
* White Packet Milk
* Future Milk Products

Workflow:

Customer Places Order
→ Admin Review Queue
→ Delivery Planning
→ Dispatch

Notes:

* No approval or rejection workflow.
* Admin review is only an operational verification checkpoint.
* Orders are immediately visible to operations.
* Customer does not wait for confirmation.

---

## 4. Value Added Product Workflow

Create a separate Value Added Product Ordering module.

Products may include:

* Paneer
* Ghee
* Butter
* Curd
* Cream
* Future Value Added Products

Workflow:

Customer Places Order
→ Pending Confirmation
→ Admin Reviews Availability
→ Confirmed OR Not Available
→ Production Planning
→ Dispatch
→ Billing

Requirements:

If confirmed:

* Customer sees Confirmed status.

If unavailable:

* Customer sees Not Available status.

This is a customer-facing confirmation workflow.
